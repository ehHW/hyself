# Chat Message Flow Manual Checklist

这份清单用于把当前自动化覆盖和真实 UI 联调对齐，优先验证聊天主链路、失败态分类和转发 / 聊天记录交互。

## 启动前提

建议直接在工作区根目录使用下面三条稳定命令：

```powershell
hyself_server/.venv/Scripts/python.exe -m uvicorn --app-dir ./hyself_server hyself_server.asgi:application --host 127.0.0.1 --port 8000 --reload --lifespan off
hyself_server/.venv/Scripts/python.exe -m celery --workdir ./hyself_server -A hyself_server.celery:app worker -l info --pool=solo -c 1
pnpm --dir ./hyself dev --host 127.0.0.1 --port 5174
```

准备数据建议：

1. 两个已启用用户 `A` / `B`
2. `A` 与 `B` 已建立好友关系，并已打开单聊
3. 一个包含 `A` / `B` / `C` 的群聊
4. `A` 拥有至少一个可发送的资源中心附件

## 自动化覆盖对齐

当前自动化已覆盖：

1. WebSocket 文本消息 ack / 接收端事件 / 未读更新
2. WebSocket 附件消息 ack / 接收端事件 / 非好友失败
3. WebSocket 好友通知 / 群通知 / 未读更新
4. WS schema / permission / business 错误分类
5. 前端 realtime 错误分类、失败态、tooltip / 菜单提示
6. REST 会话消息列表读取与隐藏成员恢复
7. forward / chat_record 关键回归

手工联调重点补足：

1. 页面级 bootstrap 与路由切换
2. composer 到消息列表的真实 UI 动线
3. 转发弹窗、聊天记录查看器、复制 / 转发交互
4. 多终端真实可见性和体验一致性
5. 离开聊天页不会残留“正在输入”状态。

### 2. 文本消息主链路

1. 从 `A -> B` 的单聊发送一条文本消息。
2. 观察发送端是否先出现本地发送态，再被服务端消息替换。
3. 在 `B` 侧确认实时收到消息、会话预览更新、未读数变化。
4. 切回 `A`，确认自己的消息不会重复残留本地占位项。

预期：

1. 发送态只短暂出现，不会卡死在 `sending`。
2. 接收端消息、会话预览、未读数一致。

### 3. 附件消息主链路

1. 从 `A -> B` 的单聊发送一张图片和一个普通文件。
2. 验证发送端本地附件消息会在 ack 后被服务端消息替换。
3. 验证接收端能看到图片预览或文件卡片。

预期：

1. 图片显示预览，文件显示名称和大小。
2. 会话最后预览与附件展示一致。

### 4. 失败态与错误分类

1. 构造一个无权限发送场景，例如解除 `A` 与 `B` 的好友关系后继续从旧单聊发送附件。
2. 构造一个格式类错误场景，例如通过开发者工具发送非法 WS payload。
3. 构造一个业务校验场景，例如回复已撤回消息。

预期：

1. 权限类失败在消息气泡下显示“无权限发送”。
2. schema 类失败在消息气泡下显示“请重试或刷新”。
3. business 类失败保留原始业务文案。
4. 红色失败按钮 tooltip 与右键菜单顶部同时显示“短提示 + 原始详细原因”。
5. 点击“重新发送”后的全局 toast 与消息区短提示分类一致。

### 5. 转发链路

1. 对单条文本消息执行转发，选择一个好友目标。
2. 对多条消息执行转发，分别测试“逐条转发”和“合并转发”。
3. 关闭再重新打开转发弹窗，确认最近转发目标仍然存在。

预期：

1. 单条转发会先弹确认框，再提交。
2. 多条转发会先选择目标，再弹“转发方式”选择框。
3. 成功后会显示“转发成功”，并清空当前多选态。

### 6. 聊天记录查看

1. 打开一条 `chat_record` 消息。
2. 在查看器中复制一条普通消息、一条图片消息。
3. 对一条带 `source_message_id` 的记录项执行转发。

预期：

1. 查看器支持递归打开聊天记录。
2. 复制成功时提示“消息已复制”，失败时提示“复制失败”。
3. 没有 `source_message_id` 的记录项会提示“该条聊天记录暂不支持转发”。

### 7. 通知与未读联动

1. 新增好友申请，确认联系人通知数变化。
2. 在群聊里触发入群申请或群系统通知，确认联系人通知页可见。
3. 打开对应聊天后确认未读数和已读同步恢复正常。

预期：

1. 好友通知、群通知、未读数三处 UI 保持一致。
2. 打开会话后不会残留错误未读。

## 建议记录模板

每次联调建议记录以下字段：

1. 操作时间
2. 使用账号
3. 场景名称
4. 实际结果
5. 是否与自动化覆盖一致
6. 若不一致，对应前端文件 / 后端接口 / WebSocket 事件
