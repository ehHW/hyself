# Hyself V2.1 发布说明

## 版本结论

当前版本已完成 V2.1 目标范围内的自动化验证、浏览器回归与人工验收，可作为 V2.1 正式发布版本使用。

本次 V2.1 的重点不是单点功能追加，而是完成一次前后端共同参与的结构升级，在保留既有业务规则的前提下，把聊天、资源中心、权限与娱乐中心收敛为更稳定、可扩展的系统边界。

## 本次发布重点

### 1. 聊天中心完成 V2.1 场景化收口

- 前端聊天页已按场景 composable 与细分 store 收口到当前 V2.1 边界
- 聊天主菜单、联系人、消息工作区、聊天记录、通知记录、巡检模式等主链路已稳定可用
- WebSocket 实时消费已统一到共享事件 envelope、dispatcher 与 runtime，聊天事件消费路径更稳定

### 2. 资源中心与聊天资产能力打通

- 资源中心文件、聊天附件与头像资产已完成当前 V2.1 所需的资产引用复用
- 同 MD5 去重、资源中心保存聊天附件、回收站恢复、上传秒传等规则已纳入统一验证范围
- 资源中心视图中的工具逻辑与响应拼装已完成当前阶段所需下沉，接口职责更清晰

### 3. 后端完成更清晰的分层收口

- chat 模块的 interfaces、application、domain、infrastructure 目录边界已物理落地
- 全局与 app-local 的 auth、validators、utils、tools 目录结构已经完成重构
- Django 主包统一为 `hyself_server.*`，旧兼容包已移除

### 4. 权限与页面落地行为完成全站校验

- RBAC 角色矩阵下，聊天、资源中心、娱乐中心、用户中心的页面落地与权限提示已覆盖浏览器回归
- 错误页、非法路由回退、娱乐中心角色落地行为已纳入 V2.1 验收
- 本地烟雾账号创建与清理流程已工具化，可重复执行浏览器联调

### 5. 消息与媒体按当前边界收口

- 图片、文件、聊天记录消息主链路已覆盖当前使用场景
- 撤回、删除、审计/巡检视角下的消息状态展示已完成本轮修正
- 统一 asset picker 与更丰富媒体能力明确延后到 V2.2，不再阻塞 V2.1 发布

## 已完成验证范围

### 自动化验证

- 前端开发服务器启动通过
- 后端 ASGI 服务启动通过
- Celery worker 启动通过
- 前端 TypeScript 检查通过
- 前端 Vitest 全量通过
- 后端关键回归测试通过：聊天、用户权限、资源中心、游戏接口

### 浏览器回归

- 聊天导航与当前会话未读联动回归通过
- RBAC 角色矩阵回归通过
- 错误页与娱乐中心落地回归通过
- 烟雾数据创建与清理链路通过

### 人工验收

- 错误页人工抽检通过
- 资源中心与上传人工抽检通过
- 娱乐中心人工抽检通过
- 聊天主链路人工联调通过

## 对使用方的影响

### 前端侧

- 聊天中心、资源中心、错误页和娱乐中心已按 V2 行为工作
- 角色无权访问时，页面会按预期回退并给出统一提示
- 浏览器联调与回归脚本可作为后续版本回归基线继续使用

### 后端侧

- 新的 Django 入口统一为 `hyself_server.asgi`、`hyself_server.celery`、`hyself_server.settings`
- 资源中心、聊天资产与权限相关接口已按新的结构组织
- 独立工具入口已补齐，可直接执行本地数据重置、烟雾数据创建/清理、回收站清理等任务

## 已知后续项

以下事项建议作为 V2.2 后续迭代继续推进，但不再阻塞本次发布：

- 统一 chat asset picker、资源中心选择器与附件入口
- 继续瘦身资源中心视图，把剩余查询编排继续下沉到 query/service
- 浏览器回归接入 CI，并统一 trace、截图、日志工件管理
